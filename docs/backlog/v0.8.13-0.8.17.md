### v0.8.17: Sweep Closeout — Ratchet & Review
**Goal**: Flip the remaining suppressed ruff rules now that the code clears them, run the full matrix, have Fable review the aggregate sweep diff, and ship — locking the sweep's gains as an enforced floor. Bookended by a **dependency audit** that holds the dependency surface to the same discipline the sweep held the code to.

**Context**: [Spoke 26](fable-sweep/26-ratchet-closeout.md) is the spec (census baselines, per-rule flip conditions, and the PLR0913/0917 decision — recommendation: keep those two off, flip the five body-complexity rules). Step 1 (basedpyright) already shipped in v0.8.13; this version handles the remainder.

**What this unlocks**: The debt cannot silently regrow — CI fails if it does. Closes the v0.8.12–v0.8.17 arc.

**Persona**: Maintainer.

- [ ] **[Spoke 26](fable-sweep/26-ratchet-closeout.md)** — re-census PLR per rule; flip cleared rules (any flip not already done in its clearing spoke); PLR0913/0917 decision (user call, options in the spoke); prune stale vulture-whitelist entries; full matrix green (`uv run pytest -m ""` · `basedpyright` · `ruff` · `pnpm --prefix web check && build` · import-linter). Effort: M | Opus
- [ ] **Fable reviews the sweep** — aggregate diff review across v0.8.13–v0.8.16: call-site completeness, behavior preservation, layer boundaries. Findings become revisions here, not silent fixes.
- [ ] **Dependency audit — every dep earns its place** *(docs-first; sibling to the v0.8.12 code audit)*

    **Story**: The maintainer (Ash) wants the dependency surface held to the same discipline the sweep just imposed on the code — every dependency **necessary** (no dead installs), on the **most modern, well-supported** option for its job, and **fully utilized** rather than shadowed by bespoke code that reimplements what the library already gives us for free. Left unaudited the tree accretes redundant or abandoned deps and hand-rolled logic a maintained library would carry.

    **Decisions**:
    - **Docs-first, like v0.8.12** — the pass produces a findings doc + work orders; any removal/swap/adoption schedules into its own revision or a later version, so the closeout ship isn't gated on refactors.
    - **Three lenses per dep** — (1) *necessity*: is it imported/used at all; (2) *modernity & support*: best-maintained option for the job, or superseded; (3) *utilization vs bespoke*: are we reimplementing what the dep already offers.
    - **Respect the supply-chain posture** — pnpm's release-age cooldown (confirmed active during the v0.8.15 dependency bump) and the transitive security-floor `overrides` in `pnpm-workspace.yaml` are documented, not fought.

    **Spec** (starting checklist — extend during the pass):
    - Scope: `pyproject.toml` (21 runtime + 12 dev) and the frontend `web/package.json` + root `package.json`.
    - *Necessity*: `knip` (frontend, already wired) + a Python unused-dep pass (e.g. `deptry`) → list zero-import deps.
    - *Modernity/support*: carry the two forward-signals the bump surfaced — **Starlette 1.3 deprecates `httpx` under its TestClient in favour of `httpx2`** (plan the migration before a future Starlette hard-removes it), and **~20 deprecated `@react-email/*` transitive subdeps arrive via `@neondatabase/auth`** (assess whether that pathway is still the right one).
    - *Utilization vs bespoke*: hunt hand-rolled logic that `tenacity` (retry/backoff), `rapidfuzz`/`jellyfish` (fuzzy/phonetic), `croniter` (schedule math), `pydantic` (validation), or `orjson` already cover.
    - Output: a `docs/backlog/` findings doc; each actionable finding becomes a scheduled work order.

    **Tests**: docs-only, no runtime change; any swap it schedules carries its own regression tests at the layer it touches. Effort: M | Executor: Opus | Status: Not Started
- [ ] **Ship** — `pyproject.toml` → `0.8.17` → `uv sync` → `pnpm --prefix web sync-api`; flip README rows for v0.8.13–v0.8.17. (Note: v0.8.13–v0.8.16 each bump `pyproject.toml` at their own deploys per version-management; 0.8.12 itself has no code artifact — the audit's tag is its docs commit.)

**Tests**: the re-enabled rules ARE the regression gate; full matrix is the acceptance bar.
