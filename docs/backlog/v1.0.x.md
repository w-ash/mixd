### v1.0.0: Maturity Gate

**Goal**: Other people log in. The version bump marks the review that says the instance is ready for them — no new features ship here by design.

**Context**: The major-version bump is reserved for "the v1.0 feature set is complete" (version-management rule); this milestone defines *complete* as a readiness review over work that already shipped, so the gate can't silently accumulate scope.

**Persona**: the invited tester (their first hour is the acceptance test); the operator (their confidence is the ship criterion).

**Key Design Decisions**:
- **No feature epics live here** — anything discovered during gate review that needs building goes to the appropriate 0.x series (or a Post-Deploy Revision of the milestone that owns it), keeping the gate a review, not a milestone-of-last-resort.
