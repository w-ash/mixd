## How to work this (read first)

You are the **orchestrator and reviewer**, not the sole implementer. Context is the scarce resource ‚Äî the reason this handoff exists.

1. **Farm each remaining task out to a subagent sized for it.** Match model and reasoning effort to the task (guidance per task below): mechanical, well-specified changes go to a cheaper/faster agent at low-to-medium effort; the protocol reshape (3e) and the data migration (4b) deserve the strongest model at high effort. One task per agent; give each agent the relevant task section below verbatim plus the file anchors ‚Äî they should not need to re-derive the design.
2. **Review every result yourself before committing.** Read the diff, check it against the task's Design and Verify sections, run the named tests, and confirm the characterization flips say what the task says they should. Subagent output is a proposal, not a commit. Push back or re-dispatch with corrections rather than patching silently around a wrong design.
3. **Sequence**: 3d ‚Üí 3e ‚Üí 4a ‚Üí 4b ‚Üí 6a ‚Üí 6b ‚Üí ship. 3e MUST precede 4a (both rewrite `lastfm/inward_resolver.py` and `spotify/cross_discovery.py`). 6a/6b are independent of 4x and can run in parallel with them if you dispatch concurrently ‚Äî but review and land 3e before anything that touches the same files.
4. **Commit per task** after your review passes (messages below; conventions: short subject, no attribution trailer). Run `uv run pytest` (full fast suite) before each commit.
