- **Current**: `build_system_prompt(today: date)` returns one cached XML-tagged block (identity/scope/domain_model/tool_habits/untrusted_content/response_format). Target (per epic): `build_system_prompt(user, library_stats, current_workflow, today)`.
- **Fetch library stats** via `GetDashboardStatsUseCase` (`src/application/use_cases/get_dashboard_stats.py`) through `execute_use_case(..., user_id=user_id)` in the route's `_build_command` (`routes/chat.py`) — the route already resolves `user_id`. Current-workflow context: when a workflow is open in the editor (pass its id from the frontend, or resolve from the request).
- **Cache discipline**: keep the stable primer as the cached prefix; append volatile per-user stats as a **later, uncached** block (Opus 4.8 cache floor = 4096 tokens). Verify `usage.cache_read_input_tokens` is nonzero on the 2nd request.
- Port couplefins `voices/` seam (single "Default" voice).

## Phase 3 — Workflow-gen tools + preview card (the flagship)

- **`workflow_schema.py`** — `build_workflow_def_schema()` walks `get_node_config_fields()` (`workflows/nodes/config_fields.py`) + `list_nodes()` (`workflows/nodes/registry.py`). **`anyOf` per node type (not `oneOf`); min/max as prose in `description` (not `minimum`/`maximum`); `additionalProperties:false`; no `strict:true`.** Derive deterministically (sorted keys) so the cached tool prefix stays byte-stable.
- **6 dispatchers** in `tool_executor.py`, registered as `ToolSpec`s: `generate_workflow_def`, `validate_workflow_def`, `save_workflow` (**`kind:"write"`**), `get_workflow`, `list_user_workflows`, `describe_node` (already shipped). Each read runs through `execute_use_case(lambda uow: …, user_id=ctx.user_id)`.
- **Validation feedback loop**: `generate_workflow_def` runs `parse_workflow_def()` (`domain/entities/workflow.py`, wrap in try/except) then `validate_workflow_def_detailed()` (`workflows/definition/validation.py`), filters via `is_validation_error()`, returns failures as an **error tool result** `{task_id, field, message}` so the model self-corrects.
- **Write-tool mechanics** (port couplefins pattern): `save_workflow`'s `dispatch` *proposes* — stores a `PendingAction` via `pending_action_store` and returns `{status:"pending_confirmation", action_id, description, details}`; its `ToolSpec.executor` *commits* on confirmation via `execute_confirmed_action` → `execute_use_case(CreateWorkflowUseCase|UpdateWorkflowUseCase, user_id)`. The route's `_handle_confirmation` (already wired) claims + executes. **The registry enforces `kind=="write" ⇔ executor is not None` at construction.**
- **Parity move**: as these tools land, move `GetWorkflowUseCase`, `ListWorkflowsUseCase`, `CreateWorkflowUseCase`, `UpdateWorkflowUseCase` out of `NOT_YET_COVERED` into the tools' `use_cases=(…)` — or `test_registry_parity.py` fails.
- **Preview card**: `WorkflowPreviewCard.tsx` embeds `web/src/components/shared/WorkflowGraph.tsx` (read-only; pass `tasks`; needs an explicit-height container). `ToolResultCard` dispatches on `tool_name === "generate_workflow_def"` (the frontend left a seam). Store slot `currentWorkflowDraft` already exists — refine-in-place keys on `action_id`; Save/Update label flips on `source`.
- **Frontend gotcha**: `CommandSearchList` is **track-specific** (hardcodes the tracks query) — a picker over playlists/workflows needs generalization or a parallel component.

Open the app → **Cmd+K** → type *"build me a chill weekend playlist"* → (Phase 3) confirm a `WorkflowDef` renders in the embedded graph → refine → **Save** → confirm it appears in the workflow list and matches the latest preview.
3. **Frontend**: `pnpm --prefix web test`, `pnpm --prefix web check`.
4. **Test-injection seam**: the fake LLM is injected via `monkeypatch.setattr(chat_route, "get_llm_client", …)` — **not** `dependency_overrides` (the LLM client resolves in the route body, not as a `Depends`, to dodge ruff B008). See `tests/integration/api/test_chat_route.py`.
