### v0.7.0: Advanced Workflow Features
**Goal**: Power-user workflow capabilities building on the visual editor foundation from v0.4.3.

**Context**: v0.4.3 delivered the core visual drag-and-drop workflow editor with node palette, configuration panel, undo/redo, and preview. This milestone adds advanced features for power users who build complex multi-branch pipelines. Workflow versioning & diff moved to v0.4.3.

#### Advanced Editor Epics

- [ ] **Sub-Flows & Node Grouping**
    - Effort: M
    - What: Group related nodes into collapsible sub-flows for complex workflows
    - Why: Large workflows (10+ nodes) become hard to navigate without grouping
    - Dependencies: v0.4.3 (Visual Editor)
    - Status: Not Started
    - Notes:
        - React Flow's `parentId` and `extent: 'parent'` for nested nodes
        - Labeled group nodes as collapsible containers
        - Expand/collapse toggle to show/hide group internals
        - Groups can be named and reused as "snippets"

- [ ] **Workflow Import/Export**
    - Effort: S
    - What: Export workflow definitions as JSON files, import from file or URL
    - Why: Share workflows between Narada instances, backup/restore
    - Dependencies: v0.4.3 (Visual Editor)
    - Status: Not Started
    - Notes:
        - Export: download WorkflowDef as `.json` file
        - Import: upload `.json` file, validate, create new workflow
        - Share via URL (encoded definition in query param for small workflows)

- [ ] **Workflow Scheduling**
    - Effort: M
    - What: Cron/interval schedules per workflow, so playlists stay fresh automatically
    - Why: Without scheduling, workflows must be manually triggered — playlists go stale
    - Dependencies: v0.5.0 (Deployment — scheduling is most useful on a running server)
    - Status: Not Started
    - Notes:
        - Prefect 3.0 native schedules: `CronSchedule`, `IntervalSchedule`, `RRuleSchedule`
        - Per-workflow schedule config: cron expression or interval, enabled/disabled toggle
        - Web UI: schedule picker on workflow detail page, next-run display on workflow list
        - API: `PUT /workflows/{id}/schedule` (CRUD for schedule config)
        - CLI: `narada workflow schedule <name> --cron "0 6 * * 1"` (or `--every 24h`)
        - Execution uses existing `RunWorkflowUseCase` + `ExecuteWorkflowRunUseCase`
        - Pairs with "Background Sync Scheduling" (below) — sync keeps data fresh, workflows transform it

- [ ] **Background Sync Scheduling**
    - Effort: M
    - What: Scheduled synchronization of play history and likes to keep source data fresh
    - Why: Workflow scheduling (above) is only useful if the underlying data is current — stale play counts produce stale playlists
    - Dependencies: Workflow Scheduling (shares scheduling infrastructure)
    - Status: Not Started
    - Notes:
        - Reuses same Prefect scheduling infrastructure as workflow scheduling
        - Sync targets: Last.fm play history, Spotify liked tracks, Spotify library
        - Configurable per-connector sync interval (e.g., Last.fm hourly, Spotify daily)
        - Uses existing use cases: `ImportListeningHistoryUseCase`, `ImportLikedTracksUseCase`
        - Web UI: sync schedule config in Settings/Connectors page
        - Complements workflow scheduling: sync data overnight → run workflows in the morning
