For strategic overview, see the [planning overview](README.md).

### v0.7.0: Workflow & Sync Scheduling
**Goal**: Automate the Weekly Curator's ritual — playlists and source data stay fresh without manual triggers.

**Context**: Without scheduling, every workflow run and data import requires a manual click. The Curator's Sunday ritual works, but it shouldn't require babysitting. Scheduling is the single highest-QoL feature for power users — it turns narada from a tool you operate into a tool that works for you. Separated from editor polish (v0.7.1) because scheduling alone is a huge win and shouldn't be blocked behind editor features.

#### Scheduling Epics

- [ ] **Workflow Scheduling**
    - Effort: M
    - What: Cron/interval schedules per workflow, so playlists stay fresh automatically
    - Why: Without scheduling, workflows must be manually triggered — playlists go stale
    - Dependencies: v0.5.0 (Deployment — scheduling is most useful on a running server)
    - Status: Not Started
    - Notes:
        - Prefect 3.0 native schedules: `CronSchedule`, `IntervalSchedule`, `RRuleSchedule`
        - Per-workflow schedule config: cron expression or interval, enabled/disabled toggle
        - Web UI: schedule picker on workflow detail page, next-run display on workflow list
        - API: `PUT /workflows/{id}/schedule` (CRUD for schedule config)
        - CLI: `narada workflow schedule <name> --cron "0 6 * * 1"` (or `--every 24h`)
        - Execution uses existing `RunWorkflowUseCase` + `ExecuteWorkflowRunUseCase`
        - Pairs with "Background Sync Scheduling" (below) — sync keeps data fresh, workflows transform it

- [ ] **Background Sync Scheduling**
    - Effort: M
    - What: Scheduled synchronization of play history and likes to keep source data fresh
    - Why: Workflow scheduling (above) is only useful if the underlying data is current — stale play counts produce stale playlists
    - Dependencies: Workflow Scheduling (shares scheduling infrastructure)
    - Status: Not Started
    - Notes:
        - Reuses same Prefect scheduling infrastructure as workflow scheduling
        - Sync targets: Last.fm play history, Spotify liked tracks, Spotify library
        - Configurable per-connector sync interval (e.g., Last.fm hourly, Spotify daily)
        - Uses existing use cases: `ImportListeningHistoryUseCase`, `ImportLikedTracksUseCase`
        - Web UI: sync schedule config in Settings/Connectors page
        - Complements workflow scheduling: sync data overnight → run workflows in the morning

### v0.7.1: Editor Polish, Templates & Playlist Browse
**Goal**: Power-user editor features and curated starting points that make workflow creation faster and more discoverable.

**Context**: v0.4.3 delivered the core visual drag-and-drop editor. This milestone adds polish: sub-flows for complex pipelines, import/export for sharing, curated templates for common patterns, and a browse UI for linking playlists. Templates also inform LLM prompt engineering in v0.8.0. Depends on v0.7.0 scheduling infrastructure being in place.

#### Editor Epics

- [ ] **Sub-Flows & Node Grouping**
    - Effort: M
    - What: Group related nodes into collapsible sub-flows for complex workflows
    - Why: Large workflows (10+ nodes) become hard to navigate without grouping
    - Dependencies: v0.4.3 (Visual Editor)
    - Status: Not Started
    - Notes:
        - React Flow's `parentId` and `extent: 'parent'` for nested nodes
        - Labeled group nodes as collapsible containers
        - Expand/collapse toggle to show/hide group internals
        - Groups can be named and reused as "snippets"

- [ ] **Workflow Import/Export**
    - Effort: S
    - What: Export workflow definitions as JSON files, import from file or URL
    - Why: Share workflows between Narada instances, backup/restore
    - Dependencies: v0.4.3 (Visual Editor)
    - Status: Not Started
    - Notes:
        - Export: download WorkflowDef as `.json` file
        - Import: upload `.json` file, validate, create new workflow
        - Share via URL (encoded definition in query param for small workflows)

- [ ] **Workflow Templates**
    - Effort: S
    - What: 5–8 curated templates for common playlist patterns
    - Why: Tinkerer starts with templates and customizes. Templates also inform LLM prompt engineering in v0.8.0.
    - Dependencies: v0.4.3 (Visual Editor), v0.7.0 (Scheduling — some templates use scheduled refresh)
    - Status: Not Started
    - Notes:
        - Template library: "Current Obsessions" (liked + 8+ plays last 30d → top 20), "Hidden Gems" (liked + 3+ plays, untouched 6mo), "Forgotten Favorites" (liked + 10+ plays all-time, 0 plays last year), "Recent Discovery Mix" (first played last 30d, 3+ plays), "Decade Deep Cuts" (liked, released 2010–2019, <5 plays), etc.
        - "New Workflow" modal offers: blank, from template, or from import
        - Each template includes a description explaining the curation logic
        - Templates ship as static WorkflowDef JSON bundled with the app
        - Template patterns directly feed v0.8.0 LLM prompt engineering — the LLM learns what "good workflows" look like from these examples

- [ ] **Browse/Search Playlists from Connector**
    - Effort: M
    - What: Browse and search user's own playlists from Spotify/Apple Music instead of paste-ID-to-link
    - Why: Paste-ID-to-link is a power-user pattern that creates unnecessary friction. Most users don't know their playlist IDs.
    - Dependencies: v0.4.4 (Connector Playlist Linking)
    - Status: Not Started
    - Notes:
        - New `list_user_playlists()` connector method (paginated, returns name + ID + track count)
        - Browse UI: list of user's playlists with search/filter, click to select
        - Replaces (or supplements) the current "paste playlist ID" flow in the link modal
        - Spotify API: `GET /me/playlists` (already have OAuth scope)
        - Apple Music: `GET /me/library/playlists` (when connector ships in v0.6.1)
        - Natural fit alongside workflow features — playlists are workflow inputs/outputs
