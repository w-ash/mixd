# Web Frontend Rules (React + TypeScript + Tailwind v4)

## Component Architecture
- **Three layers**: `ui/` (shadcn/ui Radix primitives), `shared/` (narada composites reused across pages), `pages/` (route-level, own data fetching via Tanstack Query)
- shadcn/ui is owned source in `web/src/components/ui/` ‚Äî customize freely to dark editorial aesthetic
- Keep components small. Extract to `shared/` only when reused across 2+ pages.

## State Management (No Redux/Zustand)
- **Server state**: Tanstack Query v5 ‚Äî hooks in `web/src/api/*.ts`, stale-while-revalidate, background refetch
- **URL state**: React Router search params for filters, pagination, search
- **UI state**: React `useState` / `useReducer` for modals, forms, local interactions
- **Operation progress**: `useSSE` hook + Tanstack Query for real-time operation tracking
- If cross-page state emerges, evaluate React Context before reaching for a library

## TypeScript & API Integration
- **Strict mode** always ‚Äî no `any`, no `@ts-ignore`
- API hooks export `useQuery`/`useMutation` from `web/src/api/*.ts`
- Error handling: interceptor converts API error envelope `{error: {code, message, details}}` to typed `ApiError`
- Types and hooks auto-generated by Orval from `web/openapi.json` into `web/src/api/generated/` ‚Äî never hand-edit. Regenerate with `pnpm --prefix web sync-api` (exports OpenAPI schema from FastAPI + runs Orval). Use `pnpm --prefix web generate` for Orval-only when schema hasn't changed.

## API Infrastructure
- `web/src/api/client.ts` ‚Äî `customFetch()` mutator + `ApiError` class; wraps responses into `{data, status, headers}` envelope for Orval
- `web/src/api/query-client.ts` ‚Äî `createQueryClient()` factory; retries only on `status >= 500`, error check uses `instanceof ApiError`
- `web/src/api/generated/` ‚Äî Orval output (tags-split): query/mutation hooks, MSW mock handlers, TypeScript models. Never hand-edit.

## Error Boundaries
- `react-error-boundary` wraps `<Outlet />` in PageLayout; sidebar stays outside boundary so navigation always works
- `resetKeys={[pathname]}` auto-resets error state on navigation
- `PageErrorFallback` matches EmptyState styling, uses `role="alert"` for accessibility

## Testing (Vitest + MSW)
- Co-located: `Component.tsx` ‚Üí `Component.test.tsx` (same directory)
- Hooks: `useX.ts` ‚Üí `useX.test.ts` (same directory)
- E2E: `web/e2e/*.spec.ts` (Playwright, Chromium desktop)
- Run: `pnpm --prefix web test` (Vitest) / `pnpm --prefix web test:e2e` (Playwright)
- `renderWithProviders()` from `@/test/test-utils` wraps with test QueryClient (`retry: false`, `gcTime: 0`) + `MemoryRouter` ‚Äî use for any component that uses hooks, routing, or queries
- Direct `render()` from `@testing-library/react` is fine for pure presentational components with no hooks/router
- MSW auto-generated handlers from Orval in `web/src/api/generated/**/*.msw.ts` ‚Äî pre-loaded in `setup.ts`
- Per-test API overrides: `server.use(http.get("*/api/v1/...", handler))` ‚Äî reset automatically
- `@/` path alias maps to `web/src/`
