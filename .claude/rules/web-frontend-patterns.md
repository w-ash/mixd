# Web Frontend Rules (React + TypeScript + Tailwind v4)

## Component Architecture
- **Three layers**: `ui/` (shadcn/ui Radix primitives), `shared/` (narada composites reused across pages), `pages/` (route-level, own data fetching via Tanstack Query)
- shadcn/ui is owned source in `web/src/components/ui/` ŌĆö customize freely to dark editorial aesthetic
- Keep components small. Extract to `shared/` only when reused across 2+ pages.

## State Management (No Redux/Zustand)
- **Server state**: Tanstack Query v5 ŌĆö hooks in `web/src/api/*.ts`, stale-while-revalidate, background refetch
- **URL state**: React Router search params for filters, pagination, search
- **UI state**: React `useState` / `useReducer` for modals, forms, local interactions
- **Operation progress**: `useSSE` hook + Tanstack Query for real-time operation tracking
- If cross-page state emerges, evaluate React Context before reaching for a library

## TypeScript & API Integration
- **Strict mode** always ŌĆö no `any`, no `@ts-ignore`
- API hooks export `useQuery`/`useMutation` from `web/src/api/*.ts`
- Error handling: interceptor converts API error envelope `{error: {code, message, details}}` to typed `ApiError`
- Types and hooks auto-generated by Orval from `web/openapi.json` into `web/src/api/generated/` ŌĆö never hand-edit. Regenerate with `pnpm --prefix web sync-api` (exports OpenAPI schema from FastAPI + runs Orval). Use `pnpm --prefix web generate` for Orval-only when schema hasn't changed.

## Styling
- Tailwind v4 with `@theme` tokens in `theme.css` ŌĆö CSS variables are the single source of truth
- CSS-first animations (150ms interactive, 300ms layout); Motion library only for orchestrated sequences
- Dark mode is default ŌĆö never assume light backgrounds

## API Infrastructure
- `web/src/api/client.ts` ŌĆö `customFetch()` mutator + `ApiError` class; wraps responses into `{data, status, headers}` envelope for Orval
- `web/src/api/query-client.ts` ŌĆö `createQueryClient()` factory; retries only on `status >= 500`, error check uses `instanceof ApiError`
- `web/src/api/generated/` ŌĆö Orval output (tags-split): query/mutation hooks, MSW mock handlers, TypeScript models. Never hand-edit.

## Error Boundaries
- `react-error-boundary` wraps `<Outlet />` in PageLayout; sidebar stays outside boundary so navigation always works
- `resetKeys={[pathname]}` auto-resets error state on navigation
- `PageErrorFallback` matches EmptyState styling, uses `role="alert"` for accessibility

## Testing (Vitest + MSW)
- `renderWithProviders()` from `@/test/test-utils` wraps components with test QueryClient + MemoryRouter
- MSW server from `@/test/setup` pre-loaded with Orval auto-generated handlers from `web/src/api/generated/**/*.msw.ts`
- Per-test overrides: `server.use(http.get("*/api/v1/...", handler))` ŌĆö reset automatically via `afterEach`
- Simple components (no hooks/router deps): direct `render()` from `@testing-library/react` is fine
- `@/` path alias maps to `web/src/`

## Accessibility (WCAG 2.2 AA)
- Semantic HTML: `nav`, `main`, `article`, `header` ŌĆö not div soup
- ARIA labels on all interactive elements; `aria-live` for dynamic progress updates
- Keyboard navigation: Tab order, Esc closes modals, Enter submits
- 44x44px minimum touch targets; 4.5:1 contrast ratio for text

## Anti-AI-Slop Design Principles

Narada's UI is for music connoisseurs ŌĆö the dim-lit record shop aesthetic is not decorative,
it's identity. Every surface should feel intentionally chosen, not generated by default.

### Mandatory
- **No uniform containers** ŌĆö use the 3-level depth system (inset/flat/elevated).
  If every card looks the same, the hierarchy is broken.
- **No native `<select>`** ŌĆö always use Radix Select. Native dropdowns break dark theme.
- **Entrance animations** ŌĆö page content fades up on route change. First-load lists stagger.
- **Background grain** ŌĆö the noise texture overlay is always present. It breaks flat digital monotony.
- **Asymmetric borders** ŌĆö prefer left-accent bars and bottom rules over full border boxes.
  Echoes the sidebar active state and creates visual rhythm.

### Typography Hierarchy (enforce)
- Display font (Space Grotesk): headings, buttons, nav labels, section titles
- Body font (Newsreader): descriptions, prose, metadata values
- Mono font (JetBrains Mono): ISRCs, IDs, durations, timestamps, code
- Never use Inter, Roboto, system-ui for display text

### Things That Signal "AI Generated" (avoid)
- Every container with identical `rounded-xl border bg-card p-4`
- Uniform spacing between all sections (vary the rhythm)
- `animate-pulse` skeleton loaders (use shimmer gradient)
- Text-only empty states without visual presence
- Static pages with no entrance animation
- Native browser form controls in a dark theme
