# Web Frontend Rules (React + TypeScript + Tailwind v4)

## Component Architecture
- **Three layers**: `ui/` (shadcn/ui Radix primitives), `shared/` (composites reused across pages), `pages/` (route-level, data fetching via Tanstack Query)
- shadcn/ui is owned source in `ui/` ‚Äî customize freely to dark editorial aesthetic
- Extract to `shared/` only when reused across 2+ pages

## State Management
- **Server**: Tanstack Query v5 ‚Äî stale-while-revalidate, background refetch
- **URL**: React Router search params for filters, pagination, search
- **Local UI**: `useState`/`useReducer` for modals, forms, local interactions
- **Navigation-surviving / cross-page**: Zustand ‚Äî existing stores live in `web/src/stores/` (`editor-store.ts` is the pattern: typed entry-intent state machine, v0.8.10). Reuse an existing store before creating one; a new store needs state that must outlive the route and can't ride URL params or Context.
- **Progress**: `useSSE` hook + Tanstack Query for real-time operation tracking
- No Redux; no additional state libraries beyond Zustand.

## TypeScript & API
- Strict mode always ‚Äî no `any`, no `@ts-ignore`
- Types/hooks auto-generated by Orval from `web/openapi.json` ‚Üí `web/src/api/generated/` (read-only output). Regen: `pnpm --prefix web sync-api`. Orval-only: `pnpm --prefix web generate`.
- `customFetch()` in `api/client.ts` wraps responses into `{data, status, headers}` envelope. `ApiError` class with `status`, `code`, `message`, `details`.
- `createQueryClient()` in `api/query-client.ts` ‚Äî retries only `status >= 500`

## Shared Components
- See `web-design-system.md` for the canonical component catalog and the "same action/status = same component" rule.
- Before introducing a new shared component, search `web/src/components/shared/` and `web/src/components/ui/` ‚Äî most patterns already exist.
- **Data views render through `shared/QueryStates`** (+ `shared/skeletons.tsx` primitives); **search pickers compose `shared/CommandSearchList`**. Hand-rolled loading/error/empty ladders and forked cmdk scaffolds were eliminated in v0.8.15 ‚Äî don't reintroduce them.
- **Modal dialogs use `ResponsiveDialog`** (`web/src/components/ui/responsive-dialog.tsx`) which renders as a centered modal on desktop and a bottom sheet on mobile. Don't reach for `Dialog` directly.
- **Mobile vs desktop branching** uses `useIsMobile()` from `web/src/hooks/useIsMobile.ts` ‚Äî single `lg:` breakpoint (1024px), iPad portrait counts as mobile.

## Error Boundaries
- `react-error-boundary` wraps `<Outlet />` in PageLayout; sidebar stays outside so nav always works
- `resetKeys={[pathname]}` auto-resets on navigation; `PageErrorFallback` uses `role="alert"`

## Testing (Vitest + MSW)
- Co-located: `Component.tsx` ‚Üí `Component.test.tsx`; E2E: `web/e2e/*.spec.ts` (Playwright)
- `renderWithProviders()` from `#/test/test-utils` ‚Äî wraps QueryClient (`retry: false`, `gcTime: 0`) + MemoryRouter. Use for any component with hooks/routing/queries. Plain `render()` for pure presentational.
- MSW handlers auto-generated in `api/generated/**/*.msw.ts`, pre-loaded in `setup.ts`. Per-test overrides: `server.use(http.get(...))`, reset automatically.
- `#/` alias ‚Üí `web/src/`

## Design iteration
For dev-time visual feedback (padding, alignment, responsive layout) use the Playwright MCP server ‚Äî see `web-design-review.md` for the snapshot-first loop, viewport matrix, and anti-patterns.
